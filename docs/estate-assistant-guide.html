Search Tab

🔍 Search Documents

Type any keyword — an institution name, document type, person, year, or topic — and the search engine scans your entire tokenized vault. Sensitive values like account numbers and SSNs are automatically restored from the local token registry so you see real information, not placeholder codes.

ℹ️

No Ollama needed. The Search tab works on its own, completely offline. You can use it right now without any setup.

How it works

1

Type your search term

Use specific keywords. Institution names, document types, dates, and domain terms all work well. No need for perfect spelling.

2

The engine scans your vault

It searches through every document in your Token Store — tokenized copies of your Gold and Silver vault files. The search is instant because everything is already loaded in memory.

3

Results show the real text

Every match shows a passage from the document with the original values restored — your actual account number, your real policy number, not the [ACCT_0001] placeholder. This de-tokenization happens only in memory and is never written back to disk.

Good search queries

🏦"routing number First National"01_Financial
📋"2022 federal tax return"06_Tax
🏠"flood insurance Mule property"04_Insurance
📈"Vanguard account"01_Financial
📄"trust beneficiary"07_Estate-Planning
🚗"vehicle title VIN"08_Vehicles
💡

Tip: If you get too many results, add more specific terms. If you get zero results, the document may not have been tokenized yet — run vault_tokenizer.py --vault gold --confirm on the estate laptop to index new files.

Ask Tab

🤖 Ask with AI

Ask questions in plain English and get a synthesized answer drawn directly from your vault documents. The AI finds the most relevant passages, reads them, and composes a direct response — citing which document it used.

🔒

Completely private. The AI model (Ollama) runs entirely on this machine. Your documents are never sent to any server. There is no internet connection of any kind.

How it works

1

You type a question

Ask in plain English. Be specific about what you're looking for — property name, institution, person, or document type helps narrow results.

2

Vault is searched for relevant passages

The keyword engine finds the 5 most relevant documents. The passages (with PII tokens, not real values) are assembled into context for the AI.

3

Ollama reads the passages and answers

The local AI model is given your question plus the vault passages. It writes a direct answer, naming which document it drew from. It is instructed to say "I don't see that in your vault" if the answer isn't in the documents.

4

Answer + source documents shown

The answer streams in as it's generated. Click "Sources" below the answer to see the actual vault passages the AI used, with real values restored from the registry.

Setup

Setting Up Ollama (one time)

Ollama is a free app that runs an AI model locally on your computer. You only need to set this up once. After that, it starts automatically whenever you launch Estate OS.

⏱

Setup takes about 5–10 minutes, mostly waiting for a model to download (~4 GB). You only do this once.

1

Download and install Ollama

Open a browser, go to ollama.com/download, and download the Windows installer. Run it — it installs like any normal app and starts automatically.

2

Open a terminal and pull a model

Press Win + R, type cmd, press Enter. Then run:

ollama pull mistral

This downloads the Mistral model (about 4 GB). It may take a few minutes depending on your connection. You only need to do this once.

3

Launch Estate OS

Double-click the Estate OS shortcut on your desktop. The batch file will check if Ollama is running and start it automatically if needed. The Ask tab will show Ollama running with the model name.

4

That's it

Going forward, launching Estate OS via the desktop shortcut handles everything. Ollama starts in the background, the model is ready, and the Ask tab activates automatically.

Choosing a model

Mistral is the recommended starting model — fast enough on most hardware and accurate for estate document Q&A. If you want to try alternatives:

mistral (recommended)

~4 GB. Best balance of speed and quality for factual document Q&A. Works well on the estate laptop.

ollama pull mistral

llama3

~4.7 GB. Slightly more thorough answers but slower. Good on faster hardware.

ollama pull llama3

gemma

~5 GB. Google's model. Comparable quality to Mistral.

ollama pull gemma

phi3

~2.3 GB. Microsoft's smallest model. Fastest, but less thorough on complex documents.

ollama pull phi3

You can switch models in the sidebar dropdown any time. Estate OS auto-selects the best installed model (Mistral first, then Llama 3, then others) but you can override it.

Troubleshooting

Common Issues

🔴 "Token Store not found" error on launch

The app can't find the tokenized vault documents. This means either:

1. The tokenizer hasn't been run yet. On the estate laptop, open a terminal and run:

cd C:\Users\mhhro\estate-orchestrator python behaviors\vault-tokenizer\vault_tokenizer.py --vault gold --confirm

2. The token store path is wrong in config. Check config\vault_config.json and verify the token_store path exists on this machine.

🟡 Ask tab shows "Ollama not running" even after installing

Ollama may not have started automatically after install. Open a terminal and start it manually:

ollama serve

Leave that terminal window open, then refresh the Estate OS page in your browser. Alternatively, close and reopen Estate OS using the desktop shortcut — the launch script will start Ollama automatically if it detects it's installed.

🟡 Ask tab shows "No models installed"

Ollama is running but no models have been downloaded yet. Open a terminal and run:

ollama pull mistral

This takes a few minutes (~4 GB download). Once complete, refresh the Estate OS page.

🟡 Search returns nothing for a term I know is in a document

The document may not be in the Token Store yet. The tokenizer only processes vault files that have been added since the last run. Re-run the tokenizer on the estate laptop:

python behaviors\vault-tokenizer\vault_tokenizer.py --vault gold --confirm

The tokenizer is safe to re-run — it skips files that haven't changed and only processes new or modified files.

Also check that the Cryptomator vault is unlocked (E:\ shows in Windows Explorer) before running the tokenizer.

🟡 AI answer is slow to appear

On the estate laptop, response time depends on the model size. Typical speeds:

• phi3 — ~5–10 seconds
• mistral — ~15–30 seconds
• llama3 — ~20–45 seconds

The answer streams in as it's generated, so you'll see text appearing rather than waiting for the whole thing. If it takes more than 2 minutes with no output, the model may have stalled — close and reopen Estate OS.

If speed is a persistent issue, switch to phi3 in the sidebar — it's the fastest model and still accurate for factual Q&A.

🔴 AI gives an answer that seems wrong

The AI is instructed to answer only from vault documents, but it can occasionally misread a passage or confuse similar values across documents. Always:

• Click Sources below the answer to see the actual vault passages used

• Verify the answer against the source document

• If the source passage looks wrong, the underlying vault document may be outdated — update it and re-run the tokenizer

The AI is a tool for finding information faster — not a replacement for reading the document directly.

🟡 The Cryptomator vault won't mount

The search and ask features require the vault to have been tokenized first. The tokenizer requires the Cryptomator vault to be unlocked at the time it runs. Day-to-day, once the Token Store exists, the Cryptomator vault does NOT need to be unlocked to use Estate OS — you're searching the token store, not the vault directly.

To unlock Cryptomator: open the Cryptomator app → click your vault → Unlock → enter the password. E:\ should then appear in Windows Explorer.
